Data Wrangling, Reproducibility, and Pipelines

Mini Project
Modified

March 10, 2026

Overview

In Mini Project 1, our team explored a data set and told a short story with visuals. In this second mini project, we focus on a different but equally important part of data science: turning messy data into a clean, analysis ready table.

For this project, we use the built in billboard data set from the tidyr package. The main wrangling challenge is that the weekly chart rankings are stored across many columns (wk1, wk2, wk3, and so on), which makes the data hard to analyze directly. Our goal is to build a clear pipeline that reshapes and cleans the data so that one row represents one song in one week on the chart.

This is a data wrangling project. We focus on cleaning, organizing, and documenting the data. We keep the final analysis brief and descriptive.

In Posit Cloud, we will show

  1. The rendered HTML report
  2. The complete version of this source qmd file
  3. The raw data source, which in this case is the built in billboard data set from tidyr

Step 1: Choose a data set

We chose the billboard data set from the tidyr package.

data_raw <- tidyr::billboard

Quick description of your data set

  1. What does one row represent

One row in the raw data represents one song that appeared on the Billboard chart.

  1. What makes this data set messy, incomplete, or hard to use right away

The weekly chart rankings are spread across many columns such as wk1, wk2, wk3, and so on. This wide format makes it difficult to answer questions about a song’s position over time. Also, there are many NA values in later week columns because songs often leave the chart before week 76.

  1. What might someone want to do with the cleaned version of this data

Someone might want to study how long songs stay on the chart, how rankings change over time, or which songs had the strongest chart performance.

  1. If you are using more than one raw table, what does each table contribute

We are using only one raw table for this project.

Step 2: Mini proposal

  1. What is the goal of your cleaning pipeline

The goal is to reshape the data so that each row represents one song in one chart week, with a clear week number, chart rank, and date.

  1. Who would benefit from having this cleaned table

A data analyst, a music researcher, or a student learning about tidy data would benefit from having a cleaned version of this table.

  1. List at least 4 wrangling tasks you expect to do
  • Rename unclear variables
  • Convert the chart entry date to a date format
  • Pivot the weekly ranking columns into a long format
  • Extract numeric week values from the week column names
  • Remove rows where the song was not on the chart that week
  • Create a new date variable for each week on the chart
  1. What will your final cleaned table allow someone to do more easily

The cleaned table will make it easier to track a song across weeks, summarize chart performance, and create plots or tables based on time on the chart.

  1. What is one challenge you expect to face

One challenge is deciding how to interpret the many NA values in the weekly ranking columns. In this case, those NA values are meaningful because they show that a song was no longer on the chart.

Step 4: Build your cleaning pipeline

Create a new object data that is ready for later analysis.

Our pipeline includes more than 4 meaningful wrangling actions: 1. Renaming variables 2. Converting a text date to a date object 3. Pivoting the weekly columns from wide to long 4. Extracting numeric week values 5. Removing rows with missing ranks 6. Creating a date for each week on the chart 7. Arranging rows in a clean order

data <- data_raw |>
  rename(
    artist_name = artist,
    song_title = track,
    chart_entry_date = date.entered
  ) |>
  mutate(
    chart_entry_date = as.Date(chart_entry_date, format = "%m/%d/%Y")
  ) |>
  pivot_longer(
    cols = starts_with("wk"),
    names_to = "week",
    values_to = "rank"
  ) |>
  mutate(
    week = readr::parse_number(week),
    rank = as.integer(rank),
    date_on_chart = chart_entry_date + 7 * (week - 1)
  ) |>
  filter(!is.na(rank)) |>
  arrange(artist_name, song_title, week)

Describe your main cleaning steps

  1. Step 1: We renamed variables such as artist, track, and date.entered so that the names are more descriptive and easier to read.

  2. Step 2: We converted chart_entry_date from text into a true date object so that R can recognize and work with it as a calendar date.

  3. Step 3: We used pivot_longer() to turn the many wk columns into two variables: week and rank.

  4. Step 4: We extracted the numeric week number from values like wk1 and wk2, then created a new variable called date_on_chart to show the date associated with each chart week.

  5. Optional additional step: We removed rows with missing ranks because those rows do not represent actual chart appearances.

Why your pipeline is readable

We used the pipe operator so that each step follows logically from the previous one. We also grouped related tasks together, such as renaming, reshaping, and date creation. This makes the code easier for another person to follow from top to bottom.

6.4 Handoff note

This cleaned table is designed for someone who wants to analyze chart performance over time. Each row represents one song in one week on the Billboard chart. The most important variables are artist_name, song_title, week, rank, and date_on_chart. One limitation is that this table only includes songs that made the chart and only tracks ranks while they remained on the chart. A reasonable next step would be to study how quickly songs rise or fall in ranking over time.
