Mini Project 2

Data Wrangling, Reproducibility, and Pipelines

Mini Project
Modified

March 17, 2026

Overview

In Mini Project 1, your team explored a data set and told a short story with visuals. In this second mini project, your team will focus on a different but equally important part of data science: turning messy data into a clean, analysis ready table.

  1. Identifying problems in a raw data set
  2. Building a simple, readable cleaning pipeline
  3. Checking that your cleaned data makes sense
  4. Explaining your choices in plain language

This is a data wrangling project. Focus on cleaning, organizing, and documenting your data. Do NOT do complex modeling. Keep any analysis brief and descriptive.

What you will submit

Show, in your Posit Cloud,

  1. The rendered HTML report
  2. The complete version of this source qmd file
  3. Any raw data file you used, only if it is not a built in package data set
  4. One exported cleaned data file named data_clean.csv

What you will present

A 10 minute team presentation that explains

  1. What was messy or unclear in the original data
  2. What your cleaning pipeline did
  3. Why the final cleaned table is more trustworthy and useful

Grading guide

Total 15 points:

  1. Clear purpose and clear identification of raw data issues (3 pts)
  2. Meaningful wrangling pipeline and clear documentation of steps (6 pts)
  3. Reproducible workflow, quality checks, and usable final cleaned table (3 pts)
  4. Presentation clarity and timing (3 pts)
